Derived Data Types

From the fundamental types other types can be derived by using the declaration operators. This section briefly introduces the derived types in C++

1. Arrays

           Array refer to a named list of finite number n of similar data elements. Each of the data elements can be referenced respectively by set of consecutive numbers, usually 0,1,2,3,4,5,6,7,.....,n. If the name of an array of 10 elements is X, then its elements will be referenced as shown:

                       X[0],X[1],X[2],X[3],....X[9].
Arrays can be one dimensional, two dimensional or multi dimensional.

eg:- int x[6];
     float y[4][4];

3. Pointers

       A pointer is a variable that holds a memory address. This address is usually, the location of another variable in memory. If one variable contains the address of another variable, the first variable to said to point to the second.

If a variable is going to hold a pointer , it must be declared. A pointer declaration consists of a base type, an *(asterisk), and the variable name. The general form of declaring a pointer variable is

                 type *ptr;
where type is any valid C++ type and ptr is the name of the pointer variables e.ge char *a; declares apointer to a character.

            The base type of the pointer defines what type of variables the pointer can point to. Any type of pointer can point anywhere in memory.

4. Reference

A reference is an alternative name for an object. A reference variable provides an alias for a previously defined variable. A reference declaration consists of a base type, an &(ampersand), a reference variable name equated to a variable name.
The general form of declaring a reference variable is

               type & ref-var = var-name;

where type is any valid C++ data type, ref-var is the name of reference variable that will point to variable denoted by var-name.

"There can be no reference to references, no arrays to references, and no pointers to references."
